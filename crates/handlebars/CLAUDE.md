# Handlebars 模块

[根目录](../../CLRUDE.md) > [crates](../) > **handlebars**

## 模块职责

提供轻量级的 Handlebars 模板渲染功能，支持变量替换和参数提取。用于动态生成命令、提示文本等场景。

## 入口与启动

- **主要入口**：`crates/handlebars/src/lib.rs`
- **核心函数**：`get_arguments()`、`render_template()`

## 对外接口

### 核心函数

```rust
// 提取模板中的所有变量名
pub fn get_arguments(template: &str) -> Vec<String>;

// 渲染模板，替换变量
pub fn render_template(
    template: &str,
    context: &HashMap<String, String>
) -> String;
```

### 功能特性

- **变量替换**：支持 `{{variable}}` 语法
- **参数提取**：自动提取模板中的所有变量名
- **去重**：自动去除重复的变量名
- **转义处理**：正确处理转义的括号（如 `\{{`）
- **缺失变量**：保持原始占位符不变
- **Unicode 安全**：正确处理字符和字节索引转换

## 关键依赖与配置

### 依赖项

- 无外部依赖（仅使用标准库）

### 配置要求

- 模板使用 `{{variable}}` 语法
- 使用 `{{` 转义占位符

## 数据模型

### 模板语法

- **变量占位符**：`{{variable_name}}`
- **转义**：`\{{` → `{{`
- **变量命名**：支持字母、数字、下划线

### 解析规则

- 忽略转义的占位符
- 跳过无效的占位符
- 只保留有效的变量名
- 去除重复的变量名

## 常见问题 (FAQ)

### Q: 如何转义占位符？

A: 在占位符前加反斜杠：`\{{variable}}`。

### Q: 如果变量不存在会怎样？

A: 保持原始占位符不变。

### Q: 支持哪些变量名？

A: 字母、数字、下划线的任意组合。

### Q: 性能如何？

A: 非常高效，只做字符串替换，无复杂逻辑。

### Q: 支持循环或条件吗？

A: 不支持，这是简化的模板引擎，只做变量替换。
